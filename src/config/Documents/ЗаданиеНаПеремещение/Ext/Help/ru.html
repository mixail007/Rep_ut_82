Документ "Задание на отгрузку"

Документ предназначен для учета отгрузок по машинам и создания заданий для склада. 
Данный документ имеет два вида операции:
* Расход - оформляются документы "Реализация товаров и услуг",  
* Приход - предназначен для учета поступления товаров.

Основной первичный документ при работе с оптовыми покупателями.
На основании Заказа покупателя создаются все последующие документы:
"Задание на отгрузку", "Реализация товаров и услуг", "Перемещение", "Оплата от покупателя" и т.п.

Для оперативного анализа состояния заданий на отгрузку -
 - в списке заданий принято определенное цветовое оформление как и в списке заказов покупателя :

Цвет текста - определяет наличие отгрузок по заказу:

Обычный черный текст - неотгруженные задания,
Черный текст - неотгруженные, но редактированные за последние 2 часа ,
Синий текст - есть отгрузки (реализации) по данному заданию, например РД001813

Цвет фона строки - определяет состояние заказа:

задание полностью снято например ТК010508

по заданию всё полностью отгружено  например РД001813

 На отгрузку 100% из того, что заказано например ТК010500

 Задание будет отгружен не полностью  напимер ТК010507 - нет на складе, проделен в минус или уже отгружается по другому заказу

По заданию не всё отгрузили на складе из того, что требовалось отгрузить

--------------------------------------------------------------------------------------------------------------------------------

По выбранным документам "Заказ покупателя" - формируется список еще неотгруженных товаров.
Для этих товаров - находится свободный остаток на всех складах и предлагается порядок отгрузки
по группам складов, причем после выбора 1 группы складов, товары отгружаемые с этой группы -
уже не попадают в задание на отгрузку в следующие группы складов, таким образом исключается
возможность повторной отгрузки того же самого товара - дважды.

После выбора порядка отгрузки - требуется нажать кнопку "Заполнить Товары", при этом формируется
список отгрузки товаров по конкретным складам в соответствии с выбранным порядком отгрузки, а также

Кладовщики - получают задания на отгрузку, отгружают товар в соответствие со списком складов отгрузки.

Фактическое количество отгруженного товара - заполняется в ручную или автоматически,
для 5000 склада - данные заполняются из терминала сбора данных, так же как раньше в реализации.

Все расхождения - количества, номенклатуры или склада - вносятся вручную.
Причины "Пересорта" или "недогруза" выясняются отдельно, и по возможности - устраняются.

После ввода фактического количества - заполняется закладка "Дополнительно", на которой вводятся данные по водителю,
после этого создаются документы отгрузки (Реализации). Каждый документ еще раз проверяется и проводится бухгалтером.

Только после проведения всех документов реализации - Задание на отгрузку считается выполненным.
