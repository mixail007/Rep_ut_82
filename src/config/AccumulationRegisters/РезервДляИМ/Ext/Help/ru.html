Схема резерва под Интернет магазин


Менеджер, ответственный за резервирование товара для нужд интернет-магазина (ИМ), формирует Заказы покупателей в базе ЯШТ по подразделениям.

 

В Заказах покупателей будет использоваться 2 типа контрагентов:

·        Первый тип будет использоваться для общего резервирования (брони) товара под ИМ в целом         (аналог Головного контрагента)

·        Второй тип будет использоваться для создания Заказов под конкретные заказы покупателей ИМ. Такие контрагенты будут иметь признак «Контрагент для резерва под ИМ». При проведении таких Заказов будут анализироваться сначала резервы, сделанные в целом по ИМ, затем свободные остатки ЯШТ.

Остатки ЯШТ для отображения в ИМ включают в себя фактические свободные остатки плюс остатки по резервам, сделанным в целом под ИМ.

 

Реализация:
+ 1. Для элемента справочника «Контрагенты» создается реквизит «КонтрагентДляРезерваИМ».
Для всех контрагентов, у которых будет возможность использовать товар из резерва под ИМ, необходимо будет заполнить этот реквизит.

-2. Создается регистр накопления «РезервДляИМ» с измерениями «Подразделение, Контрагент, Номенклатура» и ресурсом «Количество»
3. Доработка документа «Заказ покупателя»

3.1. При создании общего резерва под ИМ на товары документ делает приход по регистру «РезервДляИМ», остальные движения не делаем (по регистрам взаиморасчетов с контрагентами).
На одного контрагента может быть только один действующий заказ.

3.2. При проведении заказа покупателя на контрагента, у которого установлен реквизит «КонтрагентДляРезерваИМ», при проверке свободных остатков необходимо учитывать остаток по регистру «РезервДляИМ». Помимо существующих движений, создается расход в регистре «ЗаказыПокупателей», который списывает движения по общему резерву, и делает приход на себя (переписываем резерв с одного заказа на другой), а также расход по регистру «РезервДляИМ».

3.3. В первую очередь списываются товары из резерва ИМ, потом общий свободный остаток.

4. Корректировка общего резерва под ИМ делается документом «Корректировка заказа покупателя», который двигает только регистры «Заказы покупателей» и «РезервДляИМ».
5. Документ закрытие одного заказа закрывает заказ по регистрам «Заказы покупателей» и «РезервДляИМ».
