Установка общих правил

Менеджер товарной группы устанавливает "% скидки/наценки" от выбранного типа цен (Базовая или Кр.Опт)
для установки скидки - вводится "-" число, например -12,00
для наценки - вводится "+" число, например:  +4,00

 

Кроме этого, менеджер товарной группы Заполняет поле "Макс. % скидки/наценки"

 

Контроль правил клиентов

При выборе строки правила - в нижней таблице - открывается список зависимых правил клиентов.

Менеджер товарной группы может изменять эти правила, отключать, менять на фиксированный или относительный процент.

Для удобства анализа превышения скидок клиентов - максимально-разрешенной скидки - правила окрашиваются в красный цвет.

 

После изменения и записи правил клиентов - можно отправить уведомления всем менеджерам об этих изменениях.

Ввод скидок клиента менеджером по продажам.
Уведомление для менеджера товарной группы.

Задачи и События будут использоваться в системе согласования по ценам только если включена константа: [v]CогласованиеПоСкидкамКлиентов

1)      при установке Менеджером  скидок по Клиенту более макс.допустимой, при записи Правил такое правило [x]выключается и автоматически формируются задачи (по каждой товарной группе) с напоминанием для каждого менеджера товарной группы.

2)      менеджер товарной группы смотрит всплывающую задачу (через 5-10 мин), открывает эту задачу

Открывает Цены клиента и снимает  [ ]выключено либо изменяет скидку в правилах Клиента, а в заголовке задачи пишет «Да, Скидка Разрешена.» или «Нет, Скидка Запрещена!» или «Скидка Изменена на -1,5%» и ставит [v]Выполнено на задаче.

3)      Менеджеру клиента – приходит обратное уведомление во всплывающем оповещении о задачах. Для их просмотра требуется включить [v]Обратные уведомления о выполнении задач

Менеджер сразу в названии задачи видит комментарий Да / Нет / Изменено, выбирает эту задачу в списке  и нажимает кнопку [Выполнено] чтобы это уведомление больше не появлялось.

При необходимости, менеджер может открыть эту задачу и открыть из неё карточку клиента и «увидеть», что правило – включено, но с другой скидкой.

При попытке изменить скидку – правило снова выключится и появится сообщение!

Создание событий по клиентам.

Менеджер товарной группы периодически (ежемесячно) – смотрит внешний отчет №264 «Общие правила ценообразования» – и по каждому правилу (для своей товарной группы) – видит всех Клиентов и их скидки  по этому правилу.   

Все правила клиентов, которые сильно превышают "красный порог" – окрашиваются в темно-красный цвет,
скидки которые почти равны максимально-допустимой и меньше красного порога– окрашиваются в ярко-красный цвет,
величина этого поля (красный порог) можно установить с точностью до 0,1%

 

Менеджер товарной группы оценивает Клиентов и их скидки только по красным строкам и может:

1)     [х] выключить персональную скидку по Клиенту (при этом будет действовать обычная скидка)

2)     изменять скидки по всем клиентам по этому правилу – на фикс % (каскадом),
т.е. у клиента, у которого была большая скидка – изменение будет больше, а маленькая скидка изменится меньше.

3)     Установить Новую скидку для каждого Клиента – вручную (с точностью до 0.01%)

Далее, Менеджер товарной группы может сразу изменить все скидки этих клиентов самостоятельно, нажатием на кнопку [Записать новые скидки].        


И после этого нажать кнопку [Оповещения Менеджерам] при котором формируются документы «Событие» (тип: Прочее) для уведомления Менеджеров (которые указаны в столбце Менеджер), в котором будет указано, что скидка клиента по правилу № N  - изменена «Старая скидка» на «новуюСкидку».
События появятся во всплывающем списке уведомлений Менеджера, а также будут доступны всем менеджерам, работающими с данным клиентом из карточки Клиента – События по клиенту.

Внимание! «Правила ценообразования» - не периодические и установленные скидки начинают действовать сразу для всех документов (новых и старых). То есть нельзя уже будет пересчитать старый документ по «старым» скидкам!

Для периодических правил – необходимо использовать документ «Политика  ценообразования», в котором фиксировать все правила (персональные и общие).
